# Description
=============

Mount target is now <div class="richtexteditor">, not <textarea>. The editor takes over the div entirely.

<div class="richtexteditor"
     data-name="body"
     data-placeholder="Start writing…"
     data-value="{{ $post->body }}">
</div>

A hidden <input type="hidden" name="[data-name]"> is injected automatically right after the wrapper div. This is how TipTap, Quill, and every serious editor handles form submission — the hidden input syncs on every keystroke, so a plain HTML <form> POST works with no extra JS.

The editable surface is now contenteditable="div" — not a replaced textarea. 

This means:

execCommand() acts directly on real DOM nodes inside it — no round-tripping through a string.
getHTML() just clones the div's innerHTML, strips UI classes, and sanitizes. Clean and fast.
The placeholder uses the CSS :empty::before trick, which works correctly on a div.
Heading styles (h1–h6), lists, and blockquotes now actually render inside the editor, so WYSIWYG is real.

mousedown instead of click on toolbar buttons — this was a latent bug in v1. click fires after blur, so by the time the command runs the selection is already gone. mousedown + preventDefault() keeps focus (and selection) alive in the editable while the command executes.
